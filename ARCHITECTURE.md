## L1 · 光热薄膜

> 详见 [REPORT.md](REPORT.md) — 完整物理仿真与性能验证。

**L1 是整个架构的物理核心。** 它是一张 200 nm 厚的 DiSubPc·C70 有机共晶薄膜，在 242°C 的精密恒温下工作。

| 核心指标 | 数值 |
|----------|------|
| 纯光学单点积能耗 | **6 fJ** |
| 系统级单 MAC 能耗 | **17 fJ** |
| 比电子计算低 | **~60,000 倍** |
| 权重写入时间 (全矩阵) | **~30 秒** |
| 矩阵规模 | **2048 × 2048** |

**物理机制**：Query 向量以光子脉冲注入，在 1.2 ps 量子相干拍频窗口内完成与 K 矩阵（以折射率调制编码在薄膜中）的物理乘法。权重可通过微加热阵列热重写。

L1 薄膜是 L2 终端的计算引擎。每一台屋顶终端内部，都封装着一张或多张这样的薄膜。

## L2 · 小区自然光终端

**每个社区楼顶部署一套独立终端。无需电网。无需数据中心。太阳直供。**

### 2.2 能源自治

| 项目 | 数值 |
|------|------|
| 聚光面积 | 1–2 m² |
| 太阳辐照度 (地表) | ~1000 W/m² |
| 收集功率 | 1000–2000 W (峰值) |
| 终端功耗 (PID + 计算) | ~16.5 W |
| **能量自给率** | **60–120× 冗余** |
| 夜间/阴雨 | 内置相变储热单元 → 维持 242°C 8–12 小时 |

**终端不需要接入电网。** 它的 PID 恒温系统直接利用聚光的太阳热能维持 242°C 工作窗口，而非电阻加热。这彻底切断了对化石能源和电网的依赖。

### 2.3 部署场景

- **城市楼顶**：每栋住宅楼 1 台，服务全楼住户
- **乡村院落**：每户 1 台微型终端（0.5 m² 聚光）
- **移动载体**：车载/船载终端，聚光面积缩减至 0.3 m²，仍可覆盖 50 tokens/s 的推理需求
- **极地/高纬度**：日照不足时自动切换 L3 邻居互助网络获取推理接力

### 2.4 权重更新

终端的权重矩阵（即 L1 薄膜中的 K 投影）偶尔通过社区光网络静默更新：

- 更新频率：每日或每周一次（取决于模型迭代速度）
- 更新方式：夜间低负载时段，通过光纤或 Li-Fi 接收增量权重差分
- 写入能耗：16.9 fJ/pixel，全矩阵 2048×2048 约需 30 秒
- 中断容忍：更新期间推理不中断（双缓冲薄膜架构）

## L3 · 光传输与协同网络

**终端独立即能工作。但终端互联后，成为一台行星级计算机。**

### 3.1 赤道环光总线（可选主干）

```
                    ████████████████████████████████
                  ██                              ██
                ██         赤道环光总线              ██
              ██          周长 ~40,000 km             ██
                ██      纯光放大 · 无光电转换        ██
                  ██                              ██
                    ████████████████████████████████
                    
                    沿环等距部署光热计算节点
                    环上承载正在进行的推理光场
                    全球推理同步 · 单跳延迟 < 130 ms
```

**设计要点**：

| 参数 | 数值 | 说明 |
|------|------|------|
| 环周长 | ~40,000 km | 沿赤道海底光缆 + 陆地光纤 |
| 光放大间距 | ~80 km | EDFA 或拉曼放大，全光中继 |
| 环上节点数 | ~500 | 等距部署，间距约 80 km |
| 单跳延迟 | <130 ms | 光速绕地球半周 |
| 推理同步方式 | 光场叠加 | 环上同时承载多个推理会话的光信号 |

**赤道环不是必须的。** L2 终端独立即可工作。赤道环是可选的全球主干，由沿海社区自愿贡献光纤段落，逐步拼接而成。

### 3.2 邻居互助光路（网状网）

```
         terminal_A ──── 定向光束 ──── terminal_B
              │          \      /          │
              │           \    /           │
              │            \  /            │
              │             \/             │
              │             /\             │
              │            /  \            │
              │           /    \           │
              │          /      \          │
         terminal_C ──── 定向光束 ──── terminal_D
              
              纯光域路由 · 无需光电转换
              自动绕过故障/不可用节点
              去中心化网状拓扑
```

**工作原理**：

1. 每台 L2 终端配备小型光束定向器（MEMS 微镜阵列或相控阵）
2. 终端向邻居发射编码后的推理结果或权重更新（调制在 1550 nm 载波上）
3. 中间节点以全光方式转发——信号始终保持在光域，不经过光电转换
4. 网状路由协议自动发现拓扑变化，绕过不可用节点

**"逆光路网"** 的含义：与传统的"自上而下"网络（数据中心 → 用户）相反，邻居互助光路是 **横向对等网络**。没有中心节点。没有层级。每一台终端既是客户端也是服务器。

### 3.3 大气散射后备链路

当直接视距不可用（建筑物遮挡、地形阻隔、极端天气）时：

```
         terminal_A                          terminal_B
              │                                  ▲
              │  调制到不可见波段 (近红外)         │
              │  大气散射 · 非视距传播              │
              │                                  │
              │         ┌──────────┐             │
              └────────►│ 被动中继  │─────────────┘
                        │ 高空气球  │
                        │ 或反射面  │
                        └──────────┘
```

- 将信号调制到大气散射截面较大的不可见波段（如 850 nm 近红外）
- 利用大气分子和气溶胶的自然散射，经非视距路径抵达邻近节点
- 可部署高空气球或建筑反射面作为被动中继体
- **不针对任何特定环境**：此机制在晴朗、雾霾、沙尘条件下均有一定可用性（散射系数变化，但链路预算可设计冗余）

三者协同：**赤道环（洲际主干）→ 邻居光路（城域分布）→ 大气散射（最后一公里备份）**，构成无单点故障的行星级全光推理网络。

## L4 · 双面太阳帆星座

**当阳光被地球遮挡时，推理不应停止。**

### 4.1 轨道部署

```
                          ☀ 太阳
                           
                     Sail A (0°)
                        │
                        │ 120° 等距
                        │
              Sail B ───┼─── Sail C
              (120°)    │    (240°)
                        
              日心轨道 或 地月系 L4/L5 稳定点
```

三面双面太阳帆部署于 120° 等距的日心轨道或地月系统拉格朗日稳定点（L4/L5），确保**任何时候至少有一面帆面向太阳、一面帆面向地球**。

### 4.2 双面设计

```
         ╔════════════════════════════════════╗
         ║         双面太阳帆                 ║
         ╠════════════════╦═══════════════════╣
         ║   通信面 (朝外)  ║  聚焦面 (朝内)    ║
         ║                ║                   ║
         ║ · 激光星间链路  ║ · 菲涅尔聚光阵列  ║
         ║ · 对地光通信    ║ · 将阳光浓缩至     ║
         ║ · 与其他帆      ║   L1 薄膜阵列      ║
         ║   组成中继网    ║ · 维持 242°C 恒温  ║
         ║                ║ · 承载轨道推理      ║
         ╚════════════════╩═══════════════════╝
```

**两面在物理上互不干扰**：通信面指向地球和其他帆，聚焦面指向太阳。帆体通过光压实现姿态稳定——朝向太阳的一面自动获得推力，无需主动控制系统。

### 4.3 关键参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 单帆直径 | ~50 m | 聚光面可覆盖数十 kW 热功率 |
| 单帆质量 | **~4 吨** | 超薄聚酰亚胺 + DiSubPc·C70 涂层 |
| 星座总质量 | **~12 吨** | 三帆合计 |
| 轨道维持 | **光压** | 无需化学燃料 |
| 推理连续性 | **24/7/365** | 至少一帆始终沐浴阳光 |
| 对地通信延迟 | <1.3 s (地月) / <500 s (日心) | 取决于轨道选择 |
| 预估成本 | **社区众筹可及** | 见经济性分析 |

### 4.4 轨道推理的独特价值

- **"一路算力始终在线"**：地球自转不再影响计算连续性。无论地表终端处于白昼还是黑夜，轨道上的帆体始终在运行推理。
- **全球覆盖**：三帆以 120° 等距部署，任一时刻至少一束计算光束覆盖地球任一经度。
- **深空推理中继**：月球基地、深空探测器可直接接入帆体网络，无需通过地面站中转。

### 4.5 从国家项目到社区众筹

12 吨的星座质量，在传统的航天预算框架下属于中型任务（单次发射约 $50M–$200M）。但以下趋势正在改写这个方程：

- **Starship 级运力**：单次近地轨道 > 100 吨，日心轨道 > 20 吨，发射成本正在降至 $10M 以下
- **CubeSat 标准化**：帆体可以模块化制造，每个模块 10 kg，1200 个模块拼装
- **光压推进**：帆体从 LEO 到目标轨道的转移完全依赖光压，无需化学推进剂 → 发射质量大幅降低
- **社区众筹**：当单帆成本降至 ~$5M，一个有 10 万成员的全球社区每人贡献 $50 即可拥有一面帆

> **"成本可降至社区众筹级别"不是一个财务预测，而是一个物理陈述：12 吨的物质进入日心轨道所需的能量，太阳以光压的形式免费提供。人类只需要支付制造成本——而制造 50 米直径的聚酰亚胺薄膜，在工业规模下并不昂贵。**

### 四层分工

| 层级 | 角色 | 独立可用？ | 关键约束 |
|:----:|------|:---------:|----------|
| L1 | 计算引擎 | ❌ (需恒温) | 242°C ± 0.01°C |
